# 要件定義書 — Trello風タスク管理アプリ

## 1. 概要・目的

### 1.1 背景

個人やチームのタスクを「カード」として視覚的に並べ替えながら管理できる、Trello風のタスク管理Webアプリケーションを開発する。
付箋をホワイトボードに貼って動かす感覚で、直感的にタスクの状態を管理できることを目指す。

### 1.2 目的

- タスクの状態（未着手／進行中／完了 など）を一目で把握できる
- ドラッグ&ドロップ感覚でタスクを移動させ、進捗を手軽に更新できる
- 複数プロジェクト（ボード）を切り替えながらタスクを管理できる

### 1.3 本書の位置づけ

RaiseTech AIエンジニアコース 第3回講義の課題として作成する。本ドキュメントは **要件定義の目次（ハブ）** として機能し、詳細は各ドキュメントに分割して管理する。

---

## 2. 想定ユーザー

| ペルソナ | 概要 | 主な利用シーン |
| --- | --- | --- |
| 個人利用者 | 自分のやることを管理したい学生・社会人 | 勉強計画、日々のTODO管理 |
| 小規模チーム | 少人数でプロジェクトを進めるチーム（将来拡張） | プロジェクトのタスク分担、進捗共有 |

※ 初期リリースでは **個人利用** を主対象とし、チーム共有は任意機能（後続フェーズ）とする。

## 3. 機能要件（概要）

MVPとして「ユーザー管理／ボード管理／リスト管理／カード管理」の4機能を提供する。
詳細な機能一覧・ユースケースは [機能要件書](functional-requirements.md) を参照。

---

## 4. 非機能要件

| カテゴリ | 要件 |
| --- | --- |
| 性能 | 通常操作（ページ表示・カード追加等）が2秒以内に応答すること |
| セキュリティ | パスワードはハッシュ化して保存。自分のデータのみアクセス可能 |
| 可用性 | 学習用途のため高可用構成は対象外（ローカル/単一サーバで動作すれば可） |
| 保守性 | 層構造（Controller / Service / Repository 等）に基づく設計とする |
| 互換性 | モダンブラウザ（Chrome, Firefox, Edge の最新版）で動作 |

## 8. スコープ外（今回は実施しない）

- モバイルアプリ（ネイティブアプリ）対応
- リアルタイム同期（WebSocket等での他ユーザー操作の即時反映）
- オフライン対応
- 外部サービス連携（Slack通知、Googleカレンダー同期 など）
- 本番運用（スケーリング、監視、バックアップ）
